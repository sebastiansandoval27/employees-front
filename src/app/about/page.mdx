# Proceso de creación del proyecto!

## **Idea**

La idea principal del proyecto es la de organizar la jerarquía de empleados de una empresa, de tal manera que se pueda visualizar la estructura de la empresa y los empleados que la conforman.
Además de incluir un versionamiento de los empleados, de tal manera que se pueda ver el historial de cambios de un empleado en específico.
Para esto, se creará una aplicación web que permita visualizar la jerarquía de empleados y el historial de cambios de un empleado en específico.

### **Explicación modelo base de datos**

El modelo de base de datos está compuesto por dos tablas, la tabla de empleados y la tabla de jerarquía de empleados.
La tabla de empleados contiene la información de los empleados, como su nombre, su rol y su versión.
La tabla de jerarquía de empleados contiene la información de la jerarquía de los empleados, como el empleado y su manager, además de la versión de la jerarquía.

- Se presentan dos relaciones entre las tablas, la relación de empleado y jerarquía de empleado, y la relación de manager y jerarquía de empleado.

## **Base de datos**

### **Crear diagrama de base de datos y crear los modelos usando Prisma**

<br />
<br />

- **Employee Model**

- id Int @id @default(autoincrement())
- name String
- version Int
- role EmployeeRole @default(EMPLOYEE)
- employeeHierarchy EmployeeHierarchy[] @relation("EmployeeRelations")
- managerHierarchy EmployeeHierarchy[] @relation("ManagerRelations")

  <br />
  <br />

- **EmployeeHierarchy Model**

  - id Int @id @default(autoincrement())
  - version Int
  - employee Employee @relation("EmployeeRelations", fields: [employee_id], references: [id])
  - employee_id Int
  - manager Employee? @relation("ManagerRelations", fields: [manager_id], references: [id])
  - manager_id Int?
